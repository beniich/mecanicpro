// ÉTAPE 01
PRÉPARER LE SERVEUR
Mise à jour système, utilisateur dédié, dossiers, prérequis
⚠️
Avant de commencer Connecte-toi en SSH avec ton accès Root. Toutes les commandes suivantes sont à exécuter en tant que root ou avec sudo.
🖥️ SSH Connection
BASH
# Connexion SSH à ton serveur dédié
ssh root@TON_IP_SERVEUR -p 22

# Ou avec clé SSH (recommandé)
ssh -i ~/.ssh/mecapro_key root@TON_IP_SERVEUR
📦 Mise à jour système
BASH
# ── CentOS / AlmaLinux / Rocky Linux ─────────────────
dnf update -y && dnf upgrade -y

# ── Ubuntu / Debian ───────────────────────────────────
apt update && apt upgrade -y && apt dist-upgrade -y

# ── Outils essentiels ────────────────────────────────
dnf install -y curl wget git vim htop net-tools unzip \
              openssl ca-certificates gnupg2 lsof

# ── Désactiver SELinux si problèmes (optionnel) ───────
setenforce 0
sed -i 's/SELINUX=enforcing/SELINUX=permissive/' /etc/selinux/config
👤 Créer utilisateur dédié (sécurité)
BASH
# Créer un utilisateur sans privilège root pour l'app
useradd -m -s /bin/bash mecapro
passwd mecapro   # définir un mot de passe fort

# Créer les dossiers de l'application
mkdir -p /var/www/mecapro/api
mkdir -p /var/www/mecapro/blazor
mkdir -p /var/www/mecapro/logs
mkdir -p /var/www/mecapro/uploads
mkdir -p /var/backups/mecapro

# Donner les permissions
chown -R mecapro:mecapro /var/www/mecapro
chmod -R 755 /var/www/mecapro

# Dossier uploads avec permissions d'écriture
chmod -R 777 /var/www/mecapro/uploads
🔑 Configurer SSH Key (désactiver password auth)
BASH
# Sur ta machine locale — générer clé SSH
ssh-keygen -t ed25519 -C "mecapro-deploy" -f ~/.ssh/mecapro_key

# Copier la clé publique sur le serveur
ssh-copy-id -i ~/.ssh/mecapro_key.pub root@TON_IP

# Sur le serveur — désactiver authentification par mot de passe
vim /etc/ssh/sshd_config

# Modifier ces lignes :
PasswordAuthentication no
PermitRootLogin prohibit-password
PubkeyAuthentication yes
Port 2222    # changer le port par défaut (optionnel mais recommandé)

systemctl restart sshd

INSTALLER .NET 8
Runtime ASP.NET Core 8 sur Linux
🐧 CentOS / AlmaLinux / RHEL
# Ajouter le repo Microsoft
rpm -Uvh https://packages.microsoft.com/config/\
centos/8/packages-microsoft-prod.rpm

# Installer Runtime ASP.NET Core 8
dnf install -y aspnetcore-runtime-8.0

# Installer aussi le SDK (pour build sur serveur)
dnf install -y dotnet-sdk-8.0

# Vérifier
dotnet --version
# ➜ 8.0.x

dotnet --list-runtimes
# ➜ Microsoft.AspNetCore.App 8.0.x
🐧 Ubuntu 22.04 / Debian 12
# Ajouter repo Microsoft
wget https://packages.microsoft.com/config/\
ubuntu/22.04/packages-microsoft-prod.deb -O pkg.deb
dpkg -i pkg.deb
rm pkg.deb

apt update

# Installer Runtime + SDK
apt install -y aspnetcore-runtime-8.0
apt install -y dotnet-sdk-8.0

# Vérifier
dotnet --version
# ➜ 8.0.x

BASE DE DONNÉES
SQL Server ou PostgreSQL selon la RAM disponible
💡
Choix de la base de données SQL Server nécessite minimum 2 GB RAM dédiés. Si ton serveur a moins de 6 GB RAM total, utilise PostgreSQL — EF Core supporte les deux avec juste un changement de provider.
🔵 Option A — SQL Server 2022
RAM 6GB+
# Ajouter repo SQL Server
curl https://packages.microsoft.com/config/rhel/8/\
mssql-server-2022.repo > /etc/yum.repos.d/mssql.repo

dnf install -y mssql-server

# Configuration initiale
/opt/mssql/bin/mssql-conf setup
# → Choisir: 2 (Developer edition - gratuit)
# → Définir mot de passe SA fort

# Démarrer et activer
systemctl enable mssql-server --now
systemctl status mssql-server

# Installer sqlcmd
curl https://packages.microsoft.com/config/rhel/8/\
prod.repo > /etc/yum.repos.d/msprod.repo
dnf install -y mssql-tools unixODBC-devel
echo 'export PATH="$PATH:/opt/mssql-tools/bin"' >> ~/.bashrc
source ~/.bashrc

# Créer la base de données
sqlcmd -S localhost -U SA -P 'TonMotDePasse!' -Q \
  "CREATE DATABASE MecaProDb; GO"

# Créer utilisateur dédié (pas SA en prod!)
sqlcmd -S localhost -U SA -P 'TonMotDePasse!' -Q \
  "CREATE LOGIN mecapro_user WITH PASSWORD='AppPass123!';
   USE MecaProDb;
   CREATE USER mecapro_user FOR LOGIN mecapro_user;
   EXEC sp_addrolemember 'db_owner', 'mecapro_user'; GO"
🐘 Option B — PostgreSQL 16
RAM 2GB+
# Installer PostgreSQL 16
dnf install -y https://download.postgresql.org/\
pub/repos/yum/reporpms/EL-8-x86_64/\
pgdg-redhat-repo-latest.noarch.rpm
dnf -qy module disable postgresql
dnf install -y postgresql16-server postgresql16

# Initialiser et démarrer
/usr/pgsql-16/bin/postgresql-16-setup initdb
systemctl enable postgresql-16 --now

# Créer DB et utilisateur
sudo -u postgres psql << 'EOF'
CREATE USER mecapro_user WITH PASSWORD 'AppPass123!';
CREATE DATABASE mecaprodb OWNER mecapro_user;
GRANT ALL PRIVILEGES ON DATABASE mecaprodb TO mecapro_user;
\q
EOF

# Changer provider EF Core dans ton projet :
# Npgsql.EntityFrameworkCore.PostgreSQL
# Connection string :
# Host=localhost;Database=mecaprodb;
# Username=mecapro_user;Password=AppPass123!

REDIS + RABBITMQ
Cache, sessions, rate limiting, et messaging
🔴 Installer Redis 7
BASH
# CentOS/AlmaLinux
dnf install -y redis
# Ubuntu
apt install -y redis-server

# Sécuriser Redis — éditer la config
vim /etc/redis/redis.conf

# Modifier ces lignes :
bind 127.0.0.1          # écouter uniquement en local
requirepass TonMotDePasseRedisForte123!
maxmemory 512mb
maxmemory-policy allkeys-lru

# Démarrer Redis
systemctl enable redis --now
systemctl status redis

# Tester
redis-cli -a TonMotDePasseRedisForte123! ping
