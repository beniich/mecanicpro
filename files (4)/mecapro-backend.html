Auth & JWT
Authentification, tokens, refresh, 2FA et OAuth2
🔑
JWT Bearer
Access Token — 15 min TTL — RS256 asymétrique
🔄
Refresh Token
HttpOnly Cookie — 30 jours — Rotation + Revocation
📱
TOTP 2FA
Google Authenticator — OTP.NET — Backup codes
🔐 Flux d'Authentification
1
POST /api/auth/login
Email + Password envoyés en HTTPS. FluentValidation vérifie le format. Brute-force protection (5 tentatives max, lockout 15 min).
2
Vérification identité
ASP.NET Identity vérifie PBKDF2 hash. Si 2FA activé → retourne requires_2fa: true + TempToken 5 min.
3
Génération des tokens
JWT Access (RS256, 15 min) signé avec clé privée Azure Key Vault. Refresh Token opaque (SHA-256), stocké en DB hashé + HttpOnly Cookie.
4
Refresh Token Rotation
À chaque refresh, l'ancien token est révoqué et un nouveau est émis. Détection de réutilisation → révocation de toute la famille.
5
Token Blacklist (Redis)
Logout → JTI mis en blacklist Redis avec TTL = durée restante du token. Vérification à chaque requête par middleware.
🔒 JWT Service — Code
public class JwtTokenService : ITokenService { private readonly RsaSecurityKey _privateKey; private readonly JwtSettings _settings; public TokenResult GenerateAccessToken(AppUser user, IList<string> roles) { var claims = new List<Claim> { new(JwtRegisteredClaimNames.Sub, user.Id), new(JwtRegisteredClaimNames.Jti, Guid.NewGuid().ToString()), new("email", user.Email!), new("subscription", user.SubscriptionTier), new("garage_id", user.GarageId.ToString()), }; roles.ForEach(r => claims.Add(new Claim(ClaimTypes.Role, r))); var token = new JwtSecurityToken( issuer: _settings.Issuer, audience: _settings.Audience, claims: claims, expires: DateTime.UtcNow.AddMinutes(15), signingCredentials: new SigningCredentials( _privateKey, SecurityAlgorithms.RsaSha256) ); return new TokenResult( new JwtSecurityTokenHandler().WriteToken(token), token.ValidTo ); } }

Moteur d'Abonnements
Lifecycle complet, Stripe Billing, webhooks et renouvellement
🔄 Lifecycle d'un Abonnement
1
Sélection du plan → Checkout
Client choisit un plan. Redirection vers Stripe Checkout Session ou Payment Sheet (mobile). Stripe crée le Customer + Subscription.
2
Webhook Stripe → subscription.created
Notre Webhook Handler reçoit l'événement, vérifie la signature HMAC, crée la Subscription en DB, active les features du plan.
3
Renouvellement automatique
Stripe tente le débit J-3 avant expiration. En cas d'échec : retry J-1, J+3, J+7 (Dunning). Email automatique à chaque tentative.
4
Upgrade / Downgrade (Proration)
Calcul automatique de la proration Stripe. Ajustement immédiat des features. Événement subscription.updated → sync DB.
5
Annulation (Cancel at period end)
Annulation programmée à fin de période. Features restent actives jusqu'à la date. Webhook subscription.deleted → désactiver accès.
🔌 Webhook Stripe Handler
[ApiController] [Route("api/webhooks/stripe")] public class StripeWebhookController : ControllerBase { [HttpPost] public async Task<IActionResult> Handle() { var json = await new StreamReader(HttpContext.Request.Body).ReadToEndAsync(); var signature = Request.Headers["Stripe-Signature"]; // 🔐 Vérification signature HMAC var stripeEvent = EventUtility.ConstructEvent(json, signature, _webhookSecret); var result = stripeEvent.Type switch { Events.CustomerSubscriptionCreated => await _mediator.Send(new SubscriptionCreatedCommand(stripeEvent)), Events.CustomerSubscriptionUpdated => await _mediator.Send(new SubscriptionUpdatedCommand(stripeEvent)), Events.CustomerSubscriptionDeleted => await _mediator.Send(new SubscriptionCancelledCommand(stripeEvent)), Events.InvoicePaymentSucceeded => await _mediator.Send(new InvoicePaidCommand(stripeEvent)), Events.InvoicePaymentFailed => await _mediator.Send(new PaymentFailedCommand(stripeEvent)), _ => Unit.Value }; return Ok(); } }
